Introduction

Star Wars, a franchise of epic proportions with devoted fans around the entire planet. This universe filled with wonder and intrigue, created by George Lucas, has enjoyed decades of prosperity with 10 live action filmed made, 1 animated film and countless of animated tv episodes. How does the social network of the canon Star Wars universe look like? This webpage compares the social network for each film. It also takes a look at the social network the occurs in the films and TV-series as a whole. Additionally text analysis is performed to find the words that characterize each movie and TV-series, and to build a timeline of how the sentiment in the Star Wars galaxy changes over time.

Data Acquisition

To get data for the following analysis a number of Wookieepedia articles were downloaded and analyzed, in fact every Wookieepedia article that belongs to a character that has appeared in a Star Wars film or tv-episode. The scripts of every movie and tv-episodes were also downloaded from Springfield! Springfield! and will be used for some text analysis.

Data processing

To find what characters are linked together each character Wookieepedia page is analyzed for links that point to other characters. Along with these links the affiliation of each character is extracted. Before any further analysis is done the special case of Anakin Skywalker and Darth Vader has to be dealt with. Wookieepedia has only one article for them together. To split them up we do not only check who links to their page, but how they link to it. If the links name is Darth Vader then a link does not go to Anakin Skywalker and vice versa. This is not perfect but will do good enough for this analysis.

Community Detection

While preparing the data such that it could be used to build a network the affiliations of each characters were gathered. Can these affiliations shed some lights on the communities that exist within the network? Since each character can be affiliated with many organizations the asumption is made that the list of affiliations on Wookieepedia is in order of importance, i.e. that the first affiliation mentioned is the most important one. This method is rudimentary and is limiting in some ways, f.ex. can affiliations change over time so saying that a character is affiliated with the same organization always is not neccessarily correct. With this method the total amount of organizations a character can be affiliated with is 207. If we use each affiliation as a community we can calculate the modularity constant, \(M\), of that community split. The modularity will tell us something about how good a split it is. If we calculate the modularity of this community split we got a modularity constant of $$M = 0.15$$ Since \(M>0\) we know that the 207 affiliations can be used as communities, but they are not the best communities because the modularity is not high enough. Can the Louvain-algorithm for community detection gives us better communities? After running the algorithm on the network we get a total of 257 communities, and the modularity constant of that community split is $$M=0.51$$ Since we get a much higher modularity constant we can safely say that the 257 communities detected by the Louvain-algorithm are better communities than the first affiliation mentioned on the Wookieepedia pages. Because of the number of communities that exist it is very hard to visualize them or compare them to the affiliations to see if some connections exists between the two different partitions. The two following figures show the network with the node colored according to what community they belong to. Only communities with 10 members or more are colored, the others are colored white.
Nodes colored by affiliation.

Nodes colored by optimal community partition.

Comparing the two figures above we clearly see that the communities found with the Louvain-algorithm are more distinct, i.e. community members tend to be close to other members in their own community. The communities the affiliations create are more irratic and members are grouped together on the graph with members from several different communities. We can see that the Louvain-algorithm groups together many members from the same affiliation, but it also mixes the affiliations together.
